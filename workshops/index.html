What You'll Build¶

By the end of this track, you will understand how to build and run a simple but real owned AI workflow:

  • A Telegram-connected agent running on your own VPS
  • A safer Week 1 setup with basic hardening and ownership habits built in
  • A first agent shaped by clear workflow design plus SOUL.md and SKILL.md
  • A MkDocs and GitHub Pages site that publishes useful outputs from your system

The point is not to impress people with complexity. The point is to build a useful, portable system that is actually yours.

Who Should Apply¶

Beginners to agents

People who use AI chats already but have not yet built an owned agent workflow.

Non-technical professionals

People who want a concrete first system without disappearing into infrastructure sprawl.

Practical builders

People who want a working setup on their own VPS, not just a conceptual overview.

Future advanced builders

People who want the clean foundation first before moving into orchestration and larger toolchains.

Want a gentler start? Try the pre-bootcamp before joining the Intro track.

It is a lighter entry point for non-technical people and a useful warm-up for anyone planning to start the workshop.

You do not need a coding background. You need a clear use case, a willingness to follow the steps, and the discipline to finish the weekly milestones.

Programme Structure¶

Week 1

Deploy & Secure

Start with infrastructure that belongs to you. Set up your VPS, install ZeroClaw, connect Telegram, and establish the basic security habits that make the system usable in real life.

Key deliverables
  • VPS setup and SSH access
  • ZeroClaw installation and first startup
  • Telegram bot connection through config.toml
  • Basic hardening, authentication hygiene, and safer ownership practices

Week 2

Build Your Own Agent

Turn a vague idea into a bounded agent workflow. Define the job, shape the agent with SOUL.md and SKILL.md, and give it the minimum tools needed to do useful work.

Key deliverables
  • Workflow design with clear trigger, input, tools, output, and stop condition
  • A first research-style agent shaped through SOUL.md and SKILL.md
  • Web search and file-writing tool setup
  • Reusable Markdown output saved into your own filesystem

Week 3

Document & Publish

Turn saved notes into a readable public resource. Organise your Markdown, structure your MkDocs site, and publish the result through GitHub Pages.

Key deliverables
  • Clear docs/ structure and mkdocs.yml navigation
  • MkDocs Material site preview running locally
  • GitHub repository connected to an automated publishing flow
  • Live GitHub Pages URL as your proof of build

Week 4

Refine & Showcase

Use the final week to close gaps, tighten your documentation, and present what you built. This is where the system becomes coherent, explainable, and portable.

Key deliverables
  • Final troubleshooting and polish
  • Cleaner documentation and portfolio narrative
  • Short project walkthrough or cohort showcase
  • Decision on whether to deepen the foundation or continue into Builder track

After This Track¶

Foundation first

The Intro track is intentionally simple. It teaches the core architecture: Telegram, VPS, files, tools, documentation, and publishing.

Then go advanced

If you want multi-agent orchestration, larger MCP ecosystems, or automation systems like n8n and OpenCode, continue into the Builder track.

Why the split matters

Beginners need clarity more than tool sprawl. Advanced builders need broader systems and stronger operational discipline.

Next step

See the Builder track if your questions are already moving beyond a first owned agent workflow.

Frequently Asked Questions¶

Q: I'm not technical. Can I still do this?

Yes. The Intro track is designed for people who are new to agents, servers, or automation and want one concrete first system instead of endless theory.

Q: Why Telegram instead of Discord or WhatsApp?

For the Intro track, Telegram is the faster path to a working personal interface. It is easier to connect, easier to test, and closer to the phone-first workflow we want participants to experience.

Q: What if I want n8n, multi-agent orchestration, or more advanced automation?

That is Builder track territory. The Intro track teaches the foundation first, then points advanced participants toward broader systems.

Q: What happens if I cannot attend live?

All sessions are recorded. The track supports async participation, and most of the work happens on your own schedule.
